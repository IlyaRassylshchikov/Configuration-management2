# Инструмент визуализации графа зависимостей

Инструмент для визуализации графа зависимостей пакетов менеджера пакетов.

## Этап 1: Минимальный прототип с конфигурацией

### Описание

Минимальное CLI-приложение с настройкой через параметры командной строки.

### Возможности

- Настройка имени анализируемого пакета
- Указание URL репозитория или пути к тестовому репозиторию
- Режим работы с тестовым репозиторием
- Фильтрация пакетов по подстроке
- Валидация всех параметров с обработкой ошибок
- Вывод конфигурации в формате ключ-значение

### Установка

```bash
# Клонирование репозитория
git clone <URL_РЕПОЗИТОРИЯ>
cd <ИМЯ_ДИРЕКТОРИИ>

# Дать права на выполнение (для Linux/Mac)
chmod +x dependency_visualizer.py
```

### Использование

```bash
# Основной синтаксис
python dependency_visualizer.py -p <имя_пакета> -r <url_или_путь> [опции]

# Примеры
# 1. Базовый пример с URL репозитория
python dependency_visualizer.py -p numpy -r https://pypi.org/simple

# 2. Работа с тестовым репозиторием
python dependency_visualizer.py -p requests -r ./test_repo.json -t

# 3. Использование фильтра пакетов
python dependency_visualizer.py -p pandas -r https://pypi.org/simple -f "data"

# 4. Полный пример со всеми параметрами
python dependency_visualizer.py -p flask -r ./test_data/repo.json -t -f "web"

# Справка
python dependency_visualizer.py --help

# Версия
python dependency_visualizer.py --version
```

### Параметры командной строки

| Параметр | Краткая форма | Описание | Обязательный |
|----------|---------------|----------|--------------|
| `--package` | `-p` | Имя анализируемого пакета | Да |
| `--repo` | `-r` | URL репозитория или путь к файлу | Да |
| `--test-mode` | `-t` | Включить режим тестового репозитория | Нет |
| `--filter` | `-f` | Подстрока для фильтрации пакетов | Нет |
| `--help` | `-h` | Показать справку | Нет |
| `--version` | `-v` | Показать версию | Нет |

### Обработка ошибок

Приложение выполняет валидацию всех параметров:

1. **Имя пакета**:
   - Не может быть пустым
   - Максимум 255 символов

2. **URL/Путь репозитория**:
   - Не может быть пустым
   - В обычном режиме: должен начинаться с `http://`, `https://` или `git://`
   - В тестовом режиме: файл/директория должны существовать

3. **Фильтр**:
   - Максимум 100 символов

### Требования

- Python 3.6 или выше
- Стандартная библиотека Python (без дополнительных зависимостей)

### Тестирование

Для тестирования создайте тестовый файл репозитория:

```bash
# Создать тестовую директорию
mkdir -p test_data

# Создать тестовый файл
echo '{}' > test_data/test_repo.json

# Запустить с тестовым режимом
python dependency_visualizer.py -p test-package -r test_data/test_repo.json -t
```
